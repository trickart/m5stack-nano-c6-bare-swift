# Task 9: Build System

## Overview

A build pipeline using SwiftPM + toolset.json + Makefile.
`make build` automates Swift compilation through ESP image generation, and `make flash` automates flashing to the device.

## File Structure

| File | Role |
|------|------|
| `Package.swift` | SwiftPM package definition, targets & dependencies |
| `toolset.json` | Application compiler & linker flags |
| `toolset-bootloader.json` | Bootloader compiler & linker flags |
| `linker/esp32c6.ld` | Application memory map & section placement |
| `linker/bootloader.ld` | Bootloader memory map (all IRAM) |
| `Makefile` | Build, flash, and clean command automation |

## Package.swift

```swift
// swift-tools-version: 6.0
import PackageDescription

let package = Package(
    name: "esp32-c6-bare-swift",
    platforms: [.macOS(.v10_15)],
    dependencies: [
        .package(url: "https://github.com/apple/swift-mmio.git", branch: "main"),
    ],
    targets: [
        .target(
            name: "Registers",
            dependencies: [.product(name: "MMIO", package: "swift-mmio")],
            swiftSettings: [.enableExperimentalFeature("Embedded")]
        ),
        .executableTarget(
            name: "Application",
            dependencies: ["Registers"],
            swiftSettings: [
                .enableExperimentalFeature("Embedded"),
                .enableExperimentalFeature("Extern"),
            ]
        ),
    ]
)
```

### Target Structure

| Target | Type | Dependencies | Content |
|--------|------|-------------|---------|
| `Registers` | library | MMIO (swift-mmio) | Register definitions generated by SVD2Swift |
| `Application` | executable | Registers | `@main` entry point, application logic |

### Experimental Features

| Flag | Purpose |
|------|---------|
| `Embedded` | Embedded Swift mode (all targets) |
| `Extern` | Linker symbol reference via `@_extern(c, ...)` |

## toolset.json

```json
{
    "schemaVersion": "1.0",
    "swiftCompiler": {
        "extraCLIOptions": [
            "-enable-experimental-feature", "Embedded",
            "-Xfrontend", "-disable-stack-protector",
            "-use-ld=lld",
            "-nostartfiles",
            "-Xlinker", "-nostdlib",
            "-Xclang-linker", "-nostdlib",
            "-Xlinker", "-static",
            "-wmo",
            "-Osize",
            "-Xlinker", "-e", "-Xlinker", "main"
        ]
    },
    "linker": {
        "extraCLIOptions": [
            "-T", "linker/esp32c6.ld",
            "--gc-sections"
        ]
    }
}
```

### swiftCompiler Flag Descriptions

| Flag | Meaning |
|------|---------|
| `-enable-experimental-feature Embedded` | Embedded Swift (applied to all modules) |
| `-Xfrontend -disable-stack-protector` | Removes `__stack_chk_guard/fail` dependency |
| `-use-ld=lld` | Use LLD linker (RISC-V ELF support) |
| `-nostartfiles` | Do not link default startup code |
| `-Xlinker -nostdlib` / `-Xclang-linker -nostdlib` | Do not link the standard library |
| `-Xlinker -static` | Static linking |
| `-wmo` | Whole Module Optimization |
| `-Osize` | Size optimization |
| `-Xlinker -e -Xlinker main` | Set entry point to `main` (generated by `@main`) |

### Linker Flag Descriptions

| Flag | Meaning |
|------|---------|
| `-T linker/esp32c6.ld` | Custom linker script |
| `--gc-sections` | Remove unused sections |

### Note: Entry Point Configuration

`-e main` must be passed via `-Xlinker` in `swiftCompiler.extraCLIOptions`, not in `linker.extraCLIOptions`.
Options in the linker section are passed directly to LLD, but SwiftPM only defines the `main` symbol via `--defsym` during linking through swiftc.

## Makefile

```makefile
TOOLCHAINS := org.swift.630202603201a
TRIPLE := riscv32-none-none-eabi
ELF := .build/$(TRIPLE)/debug/Application
BIN := build/app.bin
BOOTLOADER_ELF := .build/$(TRIPLE)/debug/Bootloader
BOOTLOADER_BIN := build/bootloader.bin
PARTITION_BIN := build/partition-table.bin

# Add ld.lld to PATH
LLD_DIR := $(shell xcrun --toolchain $(TOOLCHAINS) -f ld.lld 2>/dev/null | xargs dirname)
export PATH := $(LLD_DIR):$(PATH)
```

### Targets

| Command | Action |
|---------|--------|
| `make build` | SwiftPM build → Generate ESP image with `Tools/elf2image.swift` |
| `make bootloader` | Build 2nd stage bootloader from `Sources/Bootloader/` |
| `make partition-table` | Generate partition table with `Tools/gen-partition-table.swift` |
| `make flash` | Build all (app + bootloader + partition table) + Flash to device |
| `make image_info` | Display information about the generated image |
| `make clean` | Delete `.build/` and `build/` |

### Flash Layout

| Offset | File | Content |
|--------|------|---------|
| `0x00000` | `build/bootloader.bin` | 2nd stage bootloader (built from `Sources/Bootloader/`) |
| `0x08000` | `build/partition-table.bin` | Partition table (generated by `Tools/gen-partition-table.swift`) |
| `0x10000` | `build/app.bin` | Swift application |

### Flash Parameters

| Parameter | Value | Meaning |
|-----------|-------|---------|
| `--flash_mode` | `dio` | Dual I/O mode |
| `--flash_freq` | `80m` | 80MHz |
| `--flash_size` | `2MB` | Flash size (NanoC6 has 4MB but 2MB is sufficient) |
| `-b` | `460800` | Write baud rate |

## Build Pipeline

```
Sources/Bootloader/ (.swift)          Sources/Application/ (.swift)
  ↓ swiftc (Embedded Swift, RISC-V)     ↓ swiftc (Embedded Swift, RISC-V)
  ↓ ld.lld (linker/bootloader.ld)       ↓ ld.lld (linker/esp32c6.ld)
  ↓ Tools/elf2image.swift                ↓ Tools/elf2image.swift
build/bootloader.bin                   build/app.bin
         ↘                               ↓
           Tools/write-flash.swift  ←  build/partition-table.bin
                    ↓                    ↑
           M5Stack NanoC6           Tools/gen-partition-table.swift
```
